# 视觉与交互质量闸门

## 艺术方向

产品主体始终是巨大、风格化的3D地球。界面采用“夜间档案馆、古地图、星图与触感纸张”的组合，但禁止通过泛滥的玻璃卡片、渐变光球、全屏模糊和无目的粒子伪造高级感。

视觉层级固定为：全球规模 → 区域群落 → 人物节点 → 文本与证据 → 返回全球。任何特效都必须支持其中一个叙事动作。

## 运动语言

- GSAP只控制相机、地球镜头和故事章节时间线。
- Motion只控制DOM面板、搜索和信息进入退出。
- 同一属性不得被两个动效系统同时控制。
- 镜头运动必须对应定位、聚焦、比较或返回全局。
- 系统启用减少动态效果时，长距离飞行改为短淡入或直接切换。

## 响应式与性能

- 高画质：桌面高性能设备，完整星点、轮廓精度和抗锯齿。
- 均衡：平板和一般桌面，限制DPR与星点数量。
- 省电：中低端移动设备，降低球体细分、轮廓采样和星点数量。
- 画质降级不得删除人物、关系、来源或筛选功能。
- 3D不可用时，文字探索必须提供完整内容与关系证据。

## 阻断条件

- 地球看起来像默认地图组件或通用后台可视化；
- 主题共鸣无法从线型和文字上与直接影响区分；
- 自动运镜不能暂停；
- 移动端无法搜索、查看人物或阅读关系来源；
- 颜色成为关系类型的唯一辨识方式；
- Canvas没有等价的DOM内容入口。
