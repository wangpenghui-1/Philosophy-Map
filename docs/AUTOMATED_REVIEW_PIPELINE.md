# 自动编辑审核流水线

## 目标

开发、内容编辑和中间复核全部自动完成。用户只接收最终审核包，并决定批准、退回、合并或部署。自动流程永远不能把自身描述为外部学者或人工同行评审。

## 状态边界

```text
candidate → edited → reviewed → 最终人工批准 → published
```

- 自动编辑器只能执行可重复、可审计的安全规范化，或把内容推进到 `reviewed`。
- `published`、合并 `main` 和部署始终是最终批准后的独立动作。
- 现有迁移记录使用 `automated-migration-review/v1` 标明自动复核身份。
- 来源不足、关系证据不足、候选内容泄漏或回归测试失败都会阻断审核包。

## 自动阶段

1. **安全编辑**：去除首尾空白、去重标量数组、同步引用涉及的来源ID，并规范自动复核身份。
2. **结构与引用**：运行Zod、引用端点、内容层级、媒体、关系方向、争议说明和候选隔离检查。
3. **证据质量**：按原典、同行评审、学术专著、专家百科和参考数据集分级；识别过于通用的引用主张及缺失责任者。
4. **覆盖治理**：验证120人发布清单、90个新增人物映射、地区与时代统计及精确公开数量。
5. **批次生产治理**：验证三批共90个生产任务、工作依赖、重试预算、来源核验与装配门禁。
6. **工程回归**：ESLint、TypeScript、生产构建、数据测试、分包预算、桌面和移动端Playwright。
7. **最终审核包**：生成JSON与Markdown，给出阻断项、警告、测试阶段、分支和提交信息。

确定性检查不能独立证明哲学主张为真。每批内容仍由执行任务的自动审阅代理进行来源阅读、跨文化偏差检查和反例检索；该过程的发现必须写回引用、内容或审核报告，不能只留在对话中。

## 命令

```bash
# 应用安全、机械的自动编辑
npm run content:edit

# 不修改文件，只验证是否仍需要自动编辑
npm run content:edit:check

# 从公开学术页面的citation metadata补全缺失作者；无法验证时保持为空并报告
npm run content:enrich-sources

# 内容、代码和构建审核；浏览器阶段标为未运行
npm run review:automated

# 自动修复后执行非浏览器审核
npm run review:fix

# 最终候选必须执行的完整审核
npm run review:full
```

报告生成到 `artifacts/review/latest.json` 和 `artifacts/review/latest.md`。该目录不提交版本库；GitHub Actions在每次PR、`main`、`codex/**`和`agent/**`推送时运行完整审核，并保存30天。

## 最终批准包

提交给用户的内容必须包括：

- 自动结论：阻断或建议进入最终审核；
- 本批实体、关系和来源数量；
- 来源与语义警告及示例条目；
- 所有工程阶段状态；
- 页面视觉与已知风险；
- 明确声明自动发布和部署处于禁止状态。

用户批准前，不创建发布提交、不合并 `main`、不部署。
