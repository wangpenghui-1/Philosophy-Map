# 内容与关系证据规范

## 发布状态

AI只能生成候选记录。候选内容只有同时通过内容编辑与学术复核，才能进入正式数据。当前样片文案均以来源摘要方式呈现，不作为原文引语。

- `candidate`：候选或机器辅助草稿，不公开；
- `edited`：完成内容编辑，等待学术复核；
- `reviewed`：完成学术复核，等待发布决定；
- `published`：满足层级要求并进入网站。

编辑元数据必须记录版本、编辑者、学术复核者、复核日期和待复核说明。内容深度与编辑状态分别记录：

- `index`：80–180字摘要、至少一个概念和一个学术来源；
- `standard`：600–1500字、主要论题与论证、代表作品、至少三个概念和两个可靠来源；
- `deep`：2500–6000字，包含背景、核心论证、争议、接受史与原典入口，至少四个来源。

## 来源优先级

1. 经校勘的原典或可靠译本；
2. 同行评审论文、学术专著；
3. 由专家撰写并持续修订的学术百科；
4. Wikidata等开放知识库只用于坐标、日期和别名候选，不能独立支撑哲学观点。

每条来源记录必须包含标题、来源类型和出版信息。URL不是必填项，以支持纸质专著和原典；DOI、ISBN或URL在存在时应记录。页码、章节或段落放入具体引用，不放在共享来源上。每个引用还必须说明它支持的主张。

## 关系类型

- **直接影响**：存在阅读、引用、注释、师承或其他可说明传播路径的证据。
- **文本传播**：强调翻译、抄写、版本流通和术语迁移，不等同于思想接受。
- **批判反驳**：后者明确以先前人物或文本为论辩对象。
- **师承传统**：说明同一传统中的继承与发展；若不存在直接接触，不得写成直接师承。
- **主题共鸣**：比较研究表明二者处理相关问题，但不主张历史传播，因此必须为非方向性关系。

证据状态与关系类型分别记录：已确证、学术支持、存在争议。存在争议的关系必须同时显示争议提示。

## 跨文化表达

- 保留仁、礼、道、空、缘起等原生概念，不用单一西方术语覆盖。
- 地理位置使用历史区域和多个活动地点，不以现代国籍替代历史身份。
- 比较需要说明比较维度，不把可比较理解为可以互相翻译。
- 不把伊斯兰哲学描述为希腊哲学通往欧洲的被动中转站。
- 不把世界哲学叙述为单一文明中心向外扩散的进步史。

## 概念与作品

- 相似术语先分别建模，通过传统内部义项说明差异；没有证据时不得默认合并为同一概念。
- 作品必须区分作者、编者、归属作者等角色，并记录归属确定性、成书年代、原文名称与语言。
- 不存在可归属作品时，应明确说明口传、散佚、集体编纂或归属争议，不虚构代表作。

## 媒体

- 优先使用授权清楚的历史肖像、照片、雕塑和手稿图像。
- 后世想象图与生成图必须标为艺术化或解释性形象，不得描述为真实肖像。
- 没有可靠形象时使用中性占位，不阻止索引条目发布。
- 每条人物媒体记录来源或署名、授权状态、真实性、呈现类型与形象说明；这些说明必须在人物页对键盘和读屏器可达。

## 覆盖矩阵

`content/knowledge/coverage/people.json` 是编辑研究队列，不是公开人物数据。名单中的候选人物必须完成来源审计、内容编辑和学术复核，才能迁入正式人物目录；覆盖不足只形成编辑待办，不自动制造人物关系。
